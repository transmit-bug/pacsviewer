# apps/server — Backend API

## Architecture

```
routes/         Hono API route handlers (one file per resource)
lib/            Shared utilities
  crud.ts       Generic CRUD router factory
  errors.ts     AppError hierarchy
  auth.ts       Token verification, permission checks
  validate.ts   Zod validation helpers
  audit.ts      Audit log writer
  audit-events.ts  Audit event type definitions
middleware/     Hono middleware
  auth.ts       authMiddleware, requireRole, requirePermission
  audit.ts      auditMiddleware — logs all mutating requests
db/             Database layer
  schema.ts     Drizzle ORM table definitions + Zod schemas
  index.ts      SQLite connection (drizzle + bun:sqlite)
  seed.ts       Dev seed data
services/       Business logic
  dicom/        DICOM parsing, storage, thumbnail generation
  pyramid.ts    Image pyramid builder
adapters/       Device adapter framework
  base.ts       Abstract BaseAdapter with lifecycle hooks
  dicom.ts      DICOM device adapter
  rest.ts       REST API device adapter
  registry.ts   Adapter registry (singleton)
  types.ts      Adapter interfaces and event emitter
dicom/          DICOM SCP server (TCP, separate from HTTP)
  server.ts     TCP server — listens on DICOM_PORT (default 11112)
  pdu.ts        PDU encoding/decoding
  services/     C-STORE, C-FIND, C-ECHO handlers
  ulp/          Upper Layer Protocol (association management)
```

## Route Pattern — createCrudRouter

Most routes use the factory in `lib/crud.ts`:

```typescript
const patientsRouter = createCrudRouter(patients, {
  name: '患者',              // Error message resource name
  queryKey: 'patients',      // Drizzle relational query key
  createSchema: insertPatientSchema,  // Zod schema from drizzle-zod
  with: { patient: true },   // Relations to include
  listWhere: (c) => { ... }, // Dynamic filters
  middleware: [[requirePermission('patients', 'create')] as any],
  routes: (router) => {
    // Custom endpoints go here
    router.get('/search', async (c) => { ... });
  },
});
```

**Generated endpoints**: `GET /` (paginated list), `GET /:id`, `POST /`, `PUT /:id`, `DELETE /:id`

**Pagination**: `?page=1&pageSize=20` — max pageSize 100.

**Custom routes** must be defined inside the `routes` callback — they are registered on the same router.

## Database

- **Engine**: SQLite via `bun:sqlite` + Drizzle ORM
- **Schema**: all tables in `db/schema.ts`, exported as named exports
- **Zod schemas**: auto-generated by `drizzle-zod` (`createInsertSchema`, `createSelectSchema`) — exported from `db/index.ts`
- **Connection**: singleton `db` instance in `db/index.ts`
- **IDs**: all primary keys are UUIDs (`text('id')`)
- **Timestamps**: ISO strings (`text('created_at')`, `text('updated_at')`)

## Error Handling

Throw errors from `lib/errors.ts` — the global handler in `index.ts` catches them:

```typescript
throw new NotFoundError('患者');    // 404
throw new ValidationError('...');   // 400
throw new UnauthorizedError();      // 401
throw new ForbiddenError();         // 403
```

All error responses follow: `{ success: false, message: string, code?: string }`

## Auth & Permissions

- `authMiddleware` — validates Bearer token, sets `c.set('user', ...)` and `c.set('userId', ...)`
- `requirePermission(resource, action)` — checks role-based permissions (e.g. `'patients'`, `'create'`)
- `requireRole(...roles)` — role name check
- Auth routes (`/api/auth/*`) are mounted **before** the auth middleware — no token required.

## Adapter Framework

Device adapters extend `BaseAdapter` with lifecycle hooks:

```
initialize(config) → onStart() → [running] → onStop() → onDestroy()
```

- `AdapterRegistry` is a singleton — use `registry.get(id)` to get adapters.
- Events emitted via `AdapterEventEmitter` (image received, status changed, etc.).
- Adapters are registered via the `/api/adapters` CRUD routes.

## DICOM SCP Server

Runs on a **separate TCP port** (default 11112, configurable via `DICOM_PORT` env var). Started in `index.ts` alongside the HTTP server.

- Handles C-STORE (image receive), C-FIND (query), C-ECHO (ping)
- PDU-level protocol implementation in `dicom/pdu.ts`
- Association management in `dicom/ulp/association.ts`
- Received images flow through `services/dicom/storage.ts` → database

## Adding a New Resource

1. Define table in `db/schema.ts` with Drizzle ORM
2. Export Zod schemas via `createInsertSchema` / `createSelectSchema`
3. Create route file in `routes/` using `createCrudRouter`
4. Register in `index.ts`: `app.route('/api/your-resource', yourRouter)`
5. Run `bun run db:push` to apply schema changes

## Gotcha

- **Custom routes in createCrudRouter**: define them inside the `routes` callback, not as separate `router.get()` calls after the factory — they won't be registered.
- **Auth middleware position**: routes mounted before `app.use('/api/*', authMiddleware)` are public. The health check and auth routes are intentionally public.
- **DICOM server is separate**: it runs on a different TCP port (11112), not on the HTTP port (3000). Don't confuse the two.
- **SQLite mode**: dev uses SQLite. For PostgreSQL production, the schema uses `text` for all fields (no native JSON/enum types) — this is intentional for portability.
- **Audit middleware**: logs all mutating requests (POST/PUT/PATCH/DELETE) to the audit_logs table. User info comes from `c.get('user')` set by auth middleware.
